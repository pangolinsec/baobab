# Data Model

Baobab's data model is built around several core entities: **TreeNodes**, **Conversations**, **Projects**, **Settings**, and **ResearchRuns**.

## TreeNode

The fundamental unit. Every message in a conversation — whether from the user or the assistant — is a `TreeNode`.

```typescript
interface TreeNode {
  // Identity
  id: string;                    // UUID
  conversationId: string;        // Parent conversation
  parentId: string | null;       // null for root nodes
  childIds: string[];            // Direct children

  // Content
  role: 'user' | 'assistant';
  content: string;               // Markdown text
  model: string;                 // Model that generated this (or "user")
  createdAt: number;             // Unix timestamp

  // Classification
  nodeType: 'standard' | 'summary' | 'merge';

  // Annotations (orthogonal to nodeType)
  starred: boolean;
  deadEnd: boolean;
  userModified: boolean;
  collapsed: boolean;
  manualPosition?: { x: number; y: number };

  // Optional metadata
  thinkingBlocks?: ThinkingBlock[];  // Reasoning / thinking blocks (see below)
  tokenUsage?: {
    inputTokens: number;
    outputTokens: number;
  };
  toolCalls?: Array<{            // Web search and other tool calls
    toolName: string;
    input: Record<string, unknown>;
    result?: string;
    searchProvider?: string;
  }>;

  // Cascade overrides
  modelOverride?: string;
  providerOverride?: string;
  systemPromptOverride?: string;
  providerId?: string;           // Which provider created this node

  // Merge metadata
  mergeSourceIds?: string[];
  mergeMode?: 'summarize' | 'full-context';

  // Node source
  source?: 'user' | 'manual';
  usedSystemPrompt?: string;     // System prompt used for this node's API call

  // Research link
  researchRunId?: string;

  // Raw API capture (debugging)
  rawApiRequest?: RawApiRequest;
  rawApiResponse?: RawApiResponse;
}
```

### ThinkingBlock

Each reasoning/thinking block is stored as a structured object rather than a plain string. This supports multiple providers (Anthropic signatures, OpenAI encrypted content), cross-node copy-paste, and fine-grained control over which blocks are active or injected.

```typescript
interface ThinkingBlock {
  id: string;
  text: string;
  providerId: string;              // 'anthropic' | 'openai' | 'plaintext'
  signature?: string;              // Anthropic thinking signature
  encryptedContent?: string;       // OpenAI/Azure encrypted reasoning content
  apiItemId?: string;              // Original API reasoning item ID (for roundtrip)
  apiSummary?: unknown;            // Raw API summary (opaque, for roundtrip re-injection)
  sourceNodeId?: string;           // Origin node (if copied/pasted)
  sourceConversationId?: string;   // Origin conversation (if copied/pasted)
  isOriginal: boolean;             // true if generated by this node's API call
  plaintextEnabled: boolean;       // Show plaintext in UI
  active: boolean;                 // false = excluded from API context
  injectAtEnd?: boolean;           // Append after last user message (OpenAI reasoning injection)
}
```

### Node types

| Type | Description |
|:-----|:------------|
| `standard` | Regular user or assistant message |
| `summary` | Generated by the summarize feature |
| `merge` | Generated by the merge feature |

A node's type is set at creation and never changes.

### Annotations

Boolean flags that can be toggled independently on any node type:

- **`starred`** — user-bookmarked message, visible in sidebar starred view
- **`deadEnd`** — marks a branch as unproductive; dims the node and all descendants. Dead-end status propagates upward to branch points.
- **`userModified`** — set when a user edits an assistant response via duplicate-and-edit
- **`collapsed`** — hides the node's subtree in the tree view

## Conversation

Groups a set of related nodes into a single conversation.

```typescript
interface Conversation {
  id: string;
  title: string;
  rootNodeId: string;
  createdAt: number;
  updatedAt: number;
  model: string;                 // Default model for new messages
  systemPrompt?: string;         // Conversation-level system prompt
  tags: string[];                // User-assigned tags
  providerId?: string;           // Default provider
  projectId?: string;            // Project association
  webSearchEnabled?: boolean;
  searchProvider?: string;
}
```

## Project

Groups related conversations and provides shared context.

```typescript
interface Project {
  id: string;
  name: string;
  description: string;
  createdAt: number;
  updatedAt: number;
  systemPrompt?: string;
  injectDescription?: boolean;
  knowledgeMode?: 'off' | 'direct' | 'agentic';
}
```

## AppSettings

Global application settings, persisted as a single row in IndexedDB.

```typescript
interface AppSettings {
  apiKey: string;
  defaultModel: string;
  theme: 'light' | 'dark';

  // Advanced API config
  thinkingEnabled: boolean;
  thinkingBudget: number;
  temperature: number;
  maxOutputTokens: number;
  topP: number | null;
  topK: number | null;

  // Reasoning
  reasoningEffort?: 'low' | 'medium' | 'high';

  // Defaults
  defaultSystemPrompt: string;
  summarizationPrompt: string;
  mergePrompt?: string;
  defaultMergeMode?: 'summarize' | 'full-context';

  // Providers
  providers: ProviderConfigData[];
  defaultProvider: string;

  // Search
  tavilyApiKey?: string;
  bingApiKey?: string;
  defaultSearchProvider?: string;

  // Pricing
  customPricing?: CustomPricingEntry[];
  captureRawApiData?: boolean;

  // Research defaults
  researchDefaultPlannerModelId?: string;
  researchDefaultPlannerProviderId?: string;
  researchDefaultSubAgentModelId?: string;
  researchDefaultSubAgentProviderId?: string;
  researchMaxSubTasks?: number;
  researchMaxToolCallsPerSubAgent?: number;
  researchMaxTotalToolCalls?: number;

  // Title generation
  autoGenerateTitles?: boolean;
  titleGenerationModel?: string;
}
```

## ResearchRun

Tracks the state of a research agent run. See `src/types/research.ts` for full type definitions including `ResearchPlan`, `ResearchSubTask`, and `ResearchProcessNode`.

## Database schema

Baobab uses [Dexie.js](https://dexie.org/) to wrap IndexedDB with a typed, promise-based API.

```typescript
class BaobabDB extends Dexie {
  conversations!: Table<Conversation, string>;
  nodes!: Table<TreeNode, string>;
  settings!: Table<AppSettings, number>;
  projects!: Table<Project, string>;
  projectFiles!: Table<LocalProjectFile, string>;
  researchRuns!: Table<ResearchRun, string>;
}
```

### Indexes

| Table | Indexes |
|:------|:--------|
| `conversations` | `id` (pk), `createdAt`, `updatedAt`, `*tags`, `projectId` |
| `nodes` | `id` (pk), `conversationId`, `parentId`, `starred`, `nodeType` |
| `settings` | `++id` (auto-increment pk) |
| `projects` | `id` (pk), `createdAt`, `updatedAt` |
| `projectFiles` | `id` (pk), `projectId`, `createdAt` |
| `researchRuns` | `id` (pk), `conversationId`, `triggerNodeId`, `status` |

### Versioning

Schema migrations are handled via Dexie's built-in versioning:

| Version | Changes |
|:--------|:--------|
| v1 | Initial schema (conversations, nodes, settings) |
| v2 | Add `nodeType`, `userModified`, `starred`, `deadEnd` to nodes |
| v3 | Add `tags` to conversations, add `starred` index |
| v4 | Add `projects` table, `projectId` to conversations |
| v5 | Add `projectFiles` table (local file storage with extracted text) |
| v6–v11 | Additional tables and field migrations |

## Data ownership

All data is owned and stored by the frontend in IndexedDB:

| Data | Storage |
|:-----|:--------|
| Conversations | IndexedDB (`conversations` table) |
| Nodes | IndexedDB (`nodes` table) |
| Settings | IndexedDB (`settings` table) |
| Projects | IndexedDB (`projects` table) |
| Project files | IndexedDB (`projectFiles` table, with extracted text) |
| Research runs | IndexedDB (`researchRuns` table) |
| Tags | IndexedDB (on Conversation objects) |
