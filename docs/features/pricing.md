# Pricing Transparency

Real-time token counting and cost estimation for every message.

## Token tracking

Every assistant response records its token usage:

- **Input tokens**: tokens sent to the model (context + prompt)
- **Output tokens**: tokens generated by the model

Token counts are displayed in the node detail panel with human-friendly formatting (e.g., "1.2K" for 1,200 tokens).

## Cost estimation

Baobab estimates the cost of each response based on the model's pricing:

```
Cost = (inputTokens × inputPrice / 1M) + (outputTokens × outputPrice / 1M)
```

Costs are displayed with adaptive decimal places — more decimal places for very small amounts.

## Pricing data sources

Pricing is resolved in priority order:

1. **Custom pricing** — user-defined overrides in Settings > Pricing
2. **Live pricing** — fetched from OpenRouter's API (covers 100+ models)
3. **Static pricing** — bundled `pricing.json` file with common model prices

### Match confidence

Each pricing lookup includes a match type indicator:

- **Exact match**: model ID matched precisely
- **Prefix match**: model ID matched a known prefix pattern (e.g., "claude-3" matches "claude-3-haiku-*")

## Settings

Configure pricing behavior in **Settings > Pricing**:

- View the current pricing table
- Add custom pricing overrides for specific models
- See which pricing source is being used for each model
