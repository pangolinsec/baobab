# Visual Indicators

Visual chips and badges on tree nodes show at a glance which settings are in effect, helping you understand the state of each node without opening the detail panel.

## Indicator types

### Model chip

Shows the effective model name when it differs from the conversation default. Appears as a small label on the node.

### System chip

Displays **"system"** when a system prompt override is active (either set directly on the node or inherited from an ancestor).

### Node type badges

| Badge | Meaning |
|:------|:--------|
| **"Summary"** | Node was generated by branch summarization |
| **"Merge"** | Node was generated by branch merge |
| **"(edited)"** | Assistant response was duplicated and edited by the user |

## Visual channel map

Each visual property on tree nodes is reserved for a single purpose to prevent collisions:

| Visual | Purpose |
|:-------|:--------|
| Border color (red) | Error state |
| Border style (dashed) | Merge request node |
| Outer ring (orange) | Selected node |
| Outer ring (green) | Reply target |
| Outer ring (blue) | Multi-selected |
| Outer ring (amber) | Search match |
| Opacity (reduced) | Dead-end branch |
| Left border (blue-gray) | Summary/merge node |
| Background tint (blue-gray) | Summary/merge node |
| Edge color (accent) | Active path |
| Edge opacity (reduced) | Dead-end edge |
| Edge style (dashed blue) | Merge overlay link |
| Pulsing animation | Streaming in progress |

### Stacking

Multiple visual treatments can apply simultaneously. For example, an error node in a dead-end branch has both a red border and reduced opacity.
